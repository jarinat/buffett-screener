# 01. Project Charter

## 1. Название проекта

**Buffett Screener**

Рабочее название для личного веб-приложения по долгосрочному отбору компаний на основе качества бизнеса и устойчивости финансовых показателей.

## 2. Проблема

Сейчас отбор компаний по "Buffett-style" критериям выполняется вручную или полу-ручным способом:

- надо искать компании самому;
- надо собирать метрики по разным источникам;
- правила отбора меняются и накапливаются;
- сложно регулярно пересматривать список;
- нет стабильного механизма оповещений по новым кандидатам;
- нет удобной истории, почему компания прошла или не прошла фильтр.

## 3. Цель продукта

Создать личный исследовательский инструмент, который:

- агрегирует данные по компаниям;
- рассчитывает настраиваемые метрики;
- позволяет собирать пользовательские "экраны" (screens);
- показывает кандидатов, удовлетворяющих правилам;
- присылает оповещения, когда появляется новый интересный кандидат;
- сохраняет объяснение результата.

## 4. Философия продукта

Это не приложение "найди акцию, которая вырастет завтра".

Это приложение "найди качественную компанию, которую разумно держать 10-20 лет, если цена и тезис остаются приемлемыми".

### Следствия из этой философии

- важнее качество данных по бизнесу, чем микро-реакция на рынок;
- важнее воспроизводимость оценки, чем избыточная "магия";
- важнее explainability, чем сложная непрозрачная модель;
- решение пользователя остается ручным;
- приложение помогает думать, а не "торгует вместо инвестора".

## 5. Целевая персона

### Primary user

Один частный инвестор в режиме private-only, который:

- инвестирует редко, обычно раз в квартал / полгода / год;
- ищет качественные компании, а не краткосрочные спекуляции;
- хочет формализовать собственную систему отбора;
- готов постепенно расширять набор метрик;
- хочет получать сигналы, а не сидеть каждый день у терминала.

## 6. Scope v1

### Входит в первую волну

- universe компаний США;
- загрузка финансовых данных;
- расчет пользовательских метрик;
- таблица компаний с фильтрами и сортировками;
- сохраненные экраны;
- карточка компании с объяснением оценки;
- email alerts;
- single-user mode без логина;
- журнал запусков screen / alert.

### Не входит в первую волну

- автоматическая торговля;
- ребалансировка портфеля;
- полноценный valuation layer как отдельный слой принятия решения;
- сложные valuation-модели DCF как обязательный элемент;
- мобильное приложение;
- multi-tenant SaaS;
- публичный доступ;
- сложные коллаборативные функции.

## 7. Принципы принятия решений

### P1. Provider-agnostic core
Ядро не должно зависеть от конкретного поставщика данных.

### P2. Rule versioning
Все правила расчета должны иметь версию.

### P3. Raw first, then derived
Сначала сохраняем сырье и нормализацию, потом считаем производные метрики.

### P4. Explain every score
По каждой компании должно быть видно:
- какие метрики были посчитаны;
- какие правила сработали;
- где не хватило данных.

### P5. Manual investment decision
Система выдает кандидатов и объяснение, но не подменяет инвестиционное решение.

## 8. Критерии успеха v1

Продукт считается полезным, если пользователь может:

1. задать screen по собственным правилам;
2. получить список компаний, отсортированный по score;
3. открыть карточку компании и понять, почему она прошла;
4. сохранить screen;
5. получать оповещения о новых совпадениях;
6. пересчитать правила после изменения метрик без переписывания всей системы.

## 9. Главный риск

Главный риск проекта - не UI и не код, а зависимость от качества и доступности фундаментальных данных.

Из этого следует архитектурное решение:
- проектировать систему так, будто смена провайдера данных неизбежна.


## 10. Зафиксированные продуктовые решения на 2026-02-28

- география первой версии: только США;
- авторизация в MVP не нужна;
- канал оповещений первой версии: email;
- стартовый набор метрик расширен ROE и PE;
- полноценный valuation layer откладывается, но PE допускается как reference / guardrail metric внутри v1.
