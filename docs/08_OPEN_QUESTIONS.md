# 08. Open Questions

Ниже список вопросов, которые остаются открытыми после уточнения стартового scope.

## Уже зафиксировано

- география v1: только США;
- auth в MVP не нужна;
- канал оповещений v1: email;
- v1 остается quality-first screener;
- ROE входит в стартовый набор метрик;
- PE входит в стартовый набор как valuation-adjacent reference / guardrail metric;
- полноценный valuation layer откладывается на следующую фазу.

## 1. Источник фундаментальных данных после Yahoo

### Q1. Какой fallback provider считать приоритетным?
Варианты:
- FMP
- Alpha Vantage
- другой платный provider

### Рабочее допущение
Первым готовим abstraction, вторым кандидатом считаем FMP.

## 2. Методика growth-классификации

### Q2. Что считать "кратковременным провалом"?
Нужно формализовать:
- сколько провалов допустимо;
- какой процент падения допустим;
- как обрабатывать отрицательную базу;
- как считать CAGR при переходе через ноль.

## 3. Пороговые значения

### Q3. Какие дефолтные пороги зафиксировать для ROE и PE?
Уже есть исходные значения:
- gross margin > 70%
- net margin > 40%

Нужно решить:
- какой дефолтный порог ROE использовать;
- будет ли PE иметь дефолтный ceiling;
- считать ли PE именно hard filter или только справочным guardrail.

### Рабочее допущение
Пороги - часть screen definition с дефолтами.

## 4. Частота обновлений

### Q4. Как часто реально обновлять данные?
Возможные режимы:
- ежедневно
- еженедельно
- вручную + планировщик раз в неделю

### Рабочее допущение
Для long-term режима достаточно weekly fundamentals refresh и ручного ad-hoc запуска.

## 5. Нужен ли LLM внутри самого продукта

### Q5. Нужен ли чат / пояснятор внутри веб-приложения на первом этапе?
С учетом твоего сценария LLM нужен скорее как средство разработки, а не как core feature продукта.

### Рабочее допущение
LLM внутри продукта в MVP не нужен.
